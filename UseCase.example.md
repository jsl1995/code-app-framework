# Use Case: Asset Tracker

> This is a **fully filled example** to show the expected level of detail in UseCase.md.

---

## Problem Statement

The facilities team currently tracks all IT assets (laptops, monitors, peripherals) in
a shared Excel spreadsheet on SharePoint. The spreadsheet is frequently out of date,
has no audit trail, and breaks when multiple people edit it simultaneously. Asset
assignments are tracked by email, which makes it impossible to see at a glance who
has what, or which assets are unassigned and available.

## Existing Solution

A shared Excel file (`IT Assets v3 FINAL (2).xlsx`) on a SharePoint document library.
What works: the column structure is well understood and accepted by the team.
What doesn't: no real-time sync, no history, no way to bulk-reassign, no mobile access.

---

## Core Capabilities — v1 Must-Haves

1. IT Admins can view, add, edit, and retire assets (laptops, monitors, peripherals)
2. IT Admins can assign an asset to an employee and record the assignment date
3. Employees can view their own currently assigned assets
4. IT Admins can search and filter assets by type, status, and assigned user
5. Full audit log of all changes (assigned, returned, retired) visible to IT Admins

## Out of Scope — v1

- Barcode / QR code scanning
- Procurement workflows or purchase orders
- Integration with ITSM ticketing (ServiceNow)
- Mobile app (desktop browser only per platform constraint)
- Reporting dashboards for IT Manager (v2)

---

## Success Criteria

- IT Admins can assign or return an asset in under 60 seconds
- Asset list loads in under 2 seconds for up to 5,000 records
- Zero data loss compared to existing spreadsheet (migration required before go-live)
- All 800 employees can view their own assets without IT involvement

**Performance expectations:** Initial load < 3s, list queries < 2s, supports 5,000 asset records

**Regulatory / compliance requirements:** GDPR — employee name and device assignment
data must remain within the EU. Power Platform environment is in West Europe region.
WCAG 2.2 AA accessibility required per internal policy.
